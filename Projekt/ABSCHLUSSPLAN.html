Abschlussplan

1. Test und Screenshot-Plan

Folgende Screenshots werden zur Dokumentation der Tests und für den Nachweis der Funktionalität und Persistenz erstellt und im Ordner Projekt/screenshots/ abgelegt:

  • Wordpress:
    • wordpress_hello_world.png (Hello World Post im Browser sichtbar)
    • wordpress_persistenz_test.png (Nach Stack Restart noch sichtbar)
  • MediaWiki:
    • mediawiki_testseite.png (Testseite erstellt im Browser sichtbar)
    • mediawiki_persistenz_test.png (Nach Stack Restart noch sichtbar)
  • Jira:
    • jira_setup_screen.png (Setup Screen im Browser sichtbar)
  • Portainer:
    • portainer_overview.png (Portainer Übersicht aller laufenden Container)

2. Vorgehen Schritt für Schritt

  1. Wordpress Stack starten: 
    cd Projekt/wordpress
docker compose up -d
  2. Im Browser: http://localhost:8080 → "Hello World" Beitrag erstellen.
  3. Screenshot speichern als wordpress_hello_world.png.
  4. Stack stoppen: 
    docker compose down
  5. Stack erneut starten: 
    docker compose up -d
  6. Im Browser prüfen → Beitrag noch da? → Screenshot wordpress_persistenz_test.png.
  7. MediaWiki Stack testen analog: 
    cd Projekt/mediawiki
docker compose up -d

    Testseite erstellen → Screenshot mediawiki_testseite.png → Persistenz testen → Screenshot mediawiki_persistenz_test.png.

  8. Jira Stack starten: 
    cd Projekt/jira
docker compose up -d

    Setup-Screen öffnen → Screenshot jira_setup_screen.png.

  9. Portainer Stack starten: 
    cd Projekt/portainer
docker compose up -d

    Portainer öffnen → Übersicht aller Container → Screenshot portainer_overview.png.

4. Finaler Git Push

  1. Alle neuen Files und Screenshots hinzufügen: 
    git add Projekt/ABSCHLUSSPLAN.html Projekt/screenshots/*
  2. Commit erstellen: 
    git commit -m "Finale Doku und Screenshots für LB Projekt"
  3. Push: 
    git push origin master
